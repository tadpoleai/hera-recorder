# Hera 采集软件

## 简述

本采集软件为各类传感器提供了统一的采集, 参数设置, 数据储存, 预览与转换接口  
该采集平台可运行于 Linux 系统, x86_64 与 arm 平台上

## 模块

### 程序

#### 数据采集/监控软件 `hera-daemon`/`hera-client`

该部分为 C/S 架构, 其中 `hera-daemon` 为服务端, `hera-client` 为基于网页的客户端  
 具体使用说明参考[采集教程](manual/record.md)

- 数据采集软件 `hera-daemon` 可根据配置连接各传感器, 进行数据的实时采集  
  通过参数配置, 可基于 IPC(共享内存接口) 将数据转发给, 对应数据接口的其他模块或者外部程序  
  其中, 传感器列表和参数可以通过客户端自由地修改  
  可将数据以单一记录文件(`.hera`)的形式储存到指定的路径(默认为`/var/hera/data`)

  可选地, 数据采集软件可带有 `hera-slam` 模块 (仅支持 Ubuntu 16.04 LTS),  
  该模块利用实时传感器数据, 进行平面地图的构建, 该地图可在客户端上实时查看以便判断采集进程

- 网页客户端 `hera-client`, 可与服务端进行交互  
  包括修改传感器配置, 控制采集, 查看/上传数据

- 小工具 `hera-daemon-finder`, 可接受局域网中开启的 `hera-daemon` 发送的 UDP 心跳包

#### 数据录制软件 `hera-record`

该程序实现的功能与 `hera-daemon` 相似, 但不支持通过客户端进行交互和数据查看  
传感器列表和参数等配置通过 json 文件给出

#### 数据重播软件 `hera-replay`

读取`.hera`记录文件, 并将传感器数据通过 IPC(共享内存接口)发出, 可用于调试其他模块或者外部程序  
具体使用说明参考[数据重播](manual/replay.md)

#### 数据转换软件 `hera-convert`

读取`.hera`记录文件, 并将传感器数据转换成与 ROS 兼容的格式并保存到 Bag 文件(`.bag`)中  
具体使用说明参考[数据转换](manual/convert.md)

#### 数据检视软件 `hera-storage-tool`

读取`.hera`记录文件, 打印记录文件的元信息, 包括采集时间, 传感器配置, 数据包内容等

### 链接库

#### 公用依赖代码库 `libhera-common`

该库为所有本项目软件的共同依赖

- ipc  
  基于共享内存的无锁环形队列模板接口
- logger  
  线程和运行时错误安全的日志类, 可打印日志到标准输出和文件, 可捕获程序运行错误和异常并打印
- third_party  
  第三方公用库, 包含 Json 解析库和枚举反射库
- utils  
  文件夹和文件系统, 线程安全队列, 时间戳的实现

#### 传感器接口库 `libhera-device`

定义了传感器的连接, 参数设置, 数据采集, 数据格式定义, 数据转换等接口的基类,
以及根据子类名字反射构造传感器的工厂类

在调用该库内部任意函数前, 可调用工厂类的`load_plugin`函数, (若不显式调用将会以默认参数调用)  
该函数会从指定路径(默认为`/usr/local/lib/hera/plugin/`)搜索具体传感器子类的动态插件, 并加载到内存中

每个传感器型号对应的动态插件分为两类, 基础版和完整版(默认加载基础版)

- 基础版 `libhera-device-plugin-<category>-<name>-<arch>.so`  
  不包含能真正连接物理传感器设备的驱动函数,
  可用于数据读写和转换传感器数据,
  一般用途只需用到基础版
- 完整版 `libhera-device-plugin-<category>-<name>-<arch>.so`
  包含传感器设备驱动全功能

在分发时, 若不需要一部分传感器型号的插件, 可将动态库文件删除

#### 记录文件读写库 `libhera-storage`

定义了读写数据记录文件 `.hera` 文件的接口,
在离线读取记录文件需要链接这个库

### 独立程序（样例）

文件夹 `standalone` 目录下, 包含有可独立于本源码仓库单独编译的程序  
该文件夹可以作为 Hera 工程在其他项目中使用时的样例, 在此不一一介绍  
特别地, 该部分程序需要将 Hera 的构建结果安装进系统目录进行编译
